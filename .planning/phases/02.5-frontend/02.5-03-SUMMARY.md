## Decisions Made
- **Embedded types in hooks:** Response type interfaces live in hook files where they're used, avoiding a separate types/ directory (as specified in the plan).
- **useOrders queryKey with status param:** The queryKey includes optional status for server-side filtering, enabling cache-per-filter without collision.
- **Triple cache invalidation:** Order mutations invalidate orders, portfolio, and positions queries on success to ensure all views reflect the new state.
- **WS at root level:** useWebSocket() is called in App.tsx so one connection lives for the app lifetime, avoiding per-page connect/disconnect cycles.
