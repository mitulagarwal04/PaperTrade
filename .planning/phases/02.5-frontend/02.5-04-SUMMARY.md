## Decisions Made

- **Named export for all components:** All 6 components use named exports for consistency with the codebase pattern
- **Generic EmptyState:** Not limited to portfolio — heading/body/action are props, making it reusable across all empty state scenarios
- **CSS-only tooltip:** StalePriceBadge uses `group-hover:block` for tooltip display — no JS, no state, no dependencies
- **max-h transition for banner:** WsReconnectBanner uses max-h/opacity transition rather than display toggle for smooth slide animation
- **No hardcoded "No positions yet" in EmptyState:** The component is generic; page-specific text will be supplied by PortfolioPage in Plan 05

## Deviations from Plan

None — plan executed as written. The Skeleton `style` prop was missing from the initial interface (plan code didn't include it in the interface despite TableSkeleton using it) and was fixed during build verification.

## Issues Encountered

- **TypeScript build error in LoadingSkeleton:** `tsc -b` (used by `npm run build`) reported `Property 'style' does not exist on type 'SkeletonProps'` because the Skeleton interface didn't include `style` despite TableSkeleton passing a `style` prop with dynamic widths. Fixed by adding `style?: React.CSSProperties` to SkeletonProps and passing it to the div element. Not caught by `tsc --noEmit` due to different config resolution.
