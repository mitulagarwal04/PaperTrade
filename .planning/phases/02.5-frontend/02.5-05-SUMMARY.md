# Phase 02.5 Plan 05: Portfolio Page with Summary Cards and Positions Table Summary

**Portfolio page with 4 data-driven summary cards (Cash Balance, Total Value, Unrealized P&L, Win Rate) and sortable positions table enriched with live WebSocket prices; Dashboard page with portfolio summary as landing.**

- **Started:** 2026-05-01T00:45:00Z

- **Tasks:** 3

## Accomplishments

- SummaryCards component renders 4 metric cards from usePortfolio hook with loading (SummaryCardsSkeleton), error (ErrorBanner with Retry), and data states
- P&L values formatted with ₹ symbol and en-IN locale, colored green (positive) or red (negative) per UI-SPEC color contract
- PositionsTable component renders sortable positions with live price enrichment from Zustand priceStore, including stale-price dimming (opacity-50)
- Sortable columns (Symbol, Qty, Avg Cost, Current Price, Market Value, P&L) with ▲▼ direction indicators and unsorted ArrowUpDown icon
- P&L columns include TrendingUp/TrendingDown icons with sign-aware coloring
- Empty portfolio shows "No positions yet" with "Place your first trade" CTA navigating to /orders
- DashboardPage renders SummaryCards as landing view with future-chart placeholder
- All TypeScript compiles cleanly (tsc --noEmit passes)

## Decisions Made

- Summary cards use responsive grid (2 cols mobile, 4 cols desktop) to prevent overflow on small screens
- Positions default sorted by market_value_inr descending (largest positions first), matching typical brokerage UX
- Stale prices from WS shown with opacity-50 dimming (not yellow badge) for subtle indication per D-12
- P&L% column is not sortable (derived value) to keep sort logic focused on API fields
