# Phase 2.5 -- UI Design Contract

> Visual and interaction contract for the PaperTrade frontend. Generated by gsd-ui-researcher, verified by gsd-ui-checker.
>
> **Product type:** Real-time trading dashboard (SaaS financial tool)
> **Aesthetic:** Serious finance. Dark backgrounds, green for gains, red for losses. No generic AI gradients, no purple/pink. Data-dense but uncluttered.
>
> **Locked decisions (from CONTEXT.md):** Dark mode only. Left sidebar collapsible + bottom tab bar on mobile. TanStack Query + Zustand. React Router v6. Portfolio summary cards + positions table. Order side panel. WS auto-reconnect exponential backoff. Stale prices: yellow badge + dimmed + tooltip.

---

## Design System

| Property | Value | Rationale |
|----------|-------|-----------|
| Component library | shadcn/ui (Radix primitives) | Industry standard 2025-2026 React. Unstyled accessible primitives. Pairs with TailwindCSS. Full dark theme support. Zero-runtime CSS variables enable light mode later. |
| Icon library | Lucide React | Standard with shadcn/ui. 1000+ icons including trading-relevant: `TrendingUp`, `TrendingDown`, `DollarSign`, `BarChart3`, `Wallet`, `ArrowUpDown`, `AlertTriangle`, `X`, `Check`, `Clock`, `RefreshCw`, `Search`, `Filter`, `Settings`, `Menu`, `PanelRightOpen`, `PanelRightClose`. |
| Font | Inter | Standard for data-dense UI dashboards. Excellent legibility at small sizes (10-12px for data tables). Variable font format reduces bundle. Open source. |
| State (server) | TanStack Query v5 | Locked per D-01. Handles caching, refetch, loading/error states for portfolio, orders, assets APIs. |
| State (client) | Zustand | Locked per D-01. Minimal store for WebSocket price state and UI preferences (sidebar collapsed, selected tab). |
| Routing | React Router v6 | Locked per D-02. 5 routes: Dashboard, Portfolio, Orders, Assets, Settings. |
| API layer | Manual typed fetch wrappers | Locked per D-03. One wrapper function per endpoint group. No codegen. |
| Build tool | Vite 5.x | Per CLAUDE.md Tech Stack. |
| Styling | TailwindCSS 3.x | Per CLAUDE.md Tech Stack. |

## Spacing Scale

All values in px. Base unit = 4px.

| Token | Value | Tailwind | Usage |
|-------|-------|----------|-------|
| 3xs | 2px | `0.5` | Divider lines, badge dot spacing |
| 2xs | 4px | `1` | Icon-to-text gap, inline padding, status dot |
| xs | 8px | `2` | Compact element spacing, table cell padding |
| sm | 12px | `3` | Form field inner padding, button padding |
| md | 16px | `4` | Default spacing between related elements, card padding |
| lg | 24px | `6` | Section padding, sidebar item gap |
| xl | 32px | `8` | Layout section gaps, card-to-card spacing |
| 2xl | 48px | `12` | Major section breaks, page section separation |
| 3xl | 64px | `16` | Page-level outer margins |

**Exceptions:** `2px` (3xs) is a justified exception to the 4px grid. Rationale: Hairline dividers, badge dots, and 1px borders require sub-grid precision for data-dense trading tables. All other values follow the 4px base grid.

Font family: `Inter`, system-ui sans-serif fallback.

**Core sizes (4):**

| Role | Size | Weight | Line Height | Tailwind Class | Usage |
|------|------|--------|-------------|----------------|-------|
| Display | 32px | 600 (Semibold) | 1.2 | `text-3xl font-semibold` | Portfolio total value, large numbers |
| Heading | 20px | 600 (Semibold) | 1.3 | `text-xl font-semibold` | Page titles |
| Subheading | 16px | 600 (Semibold) | 1.4 | `text-base font-semibold` | Section headings, card titles |
| Body | 14px | 400 (Regular) | 1.5 | `text-sm` | Paragraphs, form labels, buttons |

**Justified exceptions** (data-dense trading dashboard requires more differentiation for tables and UI chrome):

| Role | Size | Weight | Line Height | Tailwind Class | Usage |
|------|------|--------|-------------|----------------|-------|
| Data label | 12px | 600 (Semibold) | 1.4 | `text-xs font-semibold` | Table headers, compact labels |
| Data value | 14px | 400 (Regular) | 1.4 | `text-sm` | Table cell values |

**Font weights:** `400` (Regular), `600` (Semibold). Only 2 weights — use `600` for all headings and emphasis, `400` for body and data.

## Color

### Dark Theme Palette

**Proportional split:** Surface backgrounds occupy ~90% of visual area. Accent and semantic colors occupy ~10%. This keeps the interface calm and data-focused — color draws attention only where it signals meaning (gains, losses, interactive elements).

**Surface hierarchy** (dark mode, backgrounds get lighter as they move up in z-stack):

| Role | Hex | HSL | Usage | WCAG AA |
|------|-----|-----|-------|---------|
| Background (base) | `#0B0D10` | `hsl(220 20% 5%)` | Page background, outermost canvas | -- |
| Surface 1 (card) | `#13161C` | `hsl(220 18% 9%)` | Cards, sidebar, panels | -- |
| Surface 2 (elevated) | `#1A1E27` | `hsl(220 18% 13%)` | Hovered cards, dropdowns, modals | -- |
| Surface 3 (raised) | `#232833` | `hsl(220 16% 17%)` | Active/selected items, input fields | -- |

**Text hierarchy:**

| Role | Hex | HSL | Usage |
|------|-----|-----|-------|
| Primary text | `#EDEEF0` | `hsl(220 8% 93%)` | Headings, body text, high-importance data |
| Secondary text | `#A0A5B0` | `hsl(220 8% 66%)` | Labels, secondary data, descriptions |
| Muted text | `#6B7280` | `hsl(220 8% 46%)` | Placeholders, disabled text, hints |
| Brand text | `#4A90D9` | `hsl(215 65% 57%)` | Links, interactive labels, brand elements |

**Semantic colors (finance-domain specific):**

| Role | Hex | HSL | Usage |
|------|-----|-----|-------|
| Positive / Gain | `#22C55E` | `hsl(142 71% 45%)` | Green P&L values, BUY badge, positive change indicators |
| Negative / Loss | `#EF4444` | `hsl(0 84% 60%)` | Red P&L values, SELL badge, negative change indicators |
| Positive (dim) | `#166534` | `hsl(142 64% 20%)` | Background tint for gain rows, gain badges |
| Negative (dim) | `#7F1D1D` | `hsl(0 63% 31%)` | Background tint for loss rows, loss badges |
| Warning | `#F59E0B` | `hsl(38 92% 50%)` | Stale price badge, warnings, caution states |

| Info / Accent | `#3B82F6` | `hsl(217 91% 60%)` | Interactive accent, focus rings, selected state, links |

| Border (subtle) | `#1F232E` | `hsl(220 14% 15%)` | Lighter dividers, table row separators |

**Don't Hand-Roll -- Color tokens:** Define these as CSS custom properties on `:root` in `app/globals.css`. shadcn/ui uses `hsl()` variables. Follow the shadcn/ui dark theme variable naming convention (background, foreground, card, primary, secondary, muted, accent, destructive, border, ring, input) plus PaperTrade-specific tokens for positive/negative/warning.

## WebSocket Integration Contract

### Connection Lifecycle

```
Page Load -> Connect to ws://localhost:8000/ws/prices
              |
              v
           Connected (status: "connected")
              |
              v
         Receive message (type: "prices")
              |
              v
         Update Zustand store -> React re-render
              |
         +----+----+
         |         |
      Success    Error/Close
         |         |
         v         v
      Continue  Reconnect with exponential backoff:
                1s -> 2s -> 4s -> 8s -> 16s -> 30s (cap)
                Show "Reconnecting..." banner
                On success -> hide banner, reset backoff
```

### Zustand Price Store Shape

```typescript
interface PriceData {
  price: number;
  currency: string;
  source: string;
  is_stale?: boolean;
  last_updated: number; // Date.now() when received
}

interface PriceStore {
  prices: Record<string, PriceData>;
  wsStatus: "connecting" | "connected" | "disconnected" | "reconnecting";
  reconnectAttempt: number;
  updatePrices: (data: Record<string, Omit<PriceData, "last_updated">>) => void;
  setWsStatus: (status: PriceStore["wsStatus"]) => void;
}
```

### Staleness Detection

```
For each price in Zustand store:
  age = Date.now() - price.last_updated
  
  if age < STALE_THRESHOLD (default: 10s):
    render normally (bright)
  elif age < EXTENDED_STALE_THRESHOLD (default: 30s):
    dim text + tooltip ("Last updated: Xs ago")
  else:
    dim text + yellow warning badge + tooltip ("Last updated: Xs ago")
```

## Design Principles (Concise)

1. **Data first.** Numbers are the primary content. Visual hierarchy prioritizes portfolio value > P&L > position details > controls.
2. **Serious palette.** Deep dark surfaces, green/red for direction only. No playful colors, no gradients.
3. **Minimal chrome.** Thin borders, generous dark space, content breathes. No shadows on cards -- use subtle border separation.
4. **Consistent density.** Text and spacing follow the defined tokens. Never mix `text-sm` and `text-base` in the same table row.
5. **Accessible by default.** All interactive elements have focus rings. All text meets WCAG AA 4.5:1. All color coding has a non-color complement (e.g., "BUY" + green, "SELL" + red).
