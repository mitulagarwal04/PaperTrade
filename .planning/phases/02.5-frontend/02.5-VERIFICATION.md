### Gaps Summary

The frontend phase goal is structurally complete — all 5 roadmap success criteria are met at the implementation level. All required artifacts exist, are substantive, and wired to the correct data sources. However, there are 4 high-severity and 1 critical-severity unfixed issues from the code review that must be addressed:

1. **CRITICAL: Extraneous dependencies (CR-01):** 98 runtime packages used by shadcn components are not declared in package.json. Running `npm ci` will fail. Fix: add all @radix-ui/* deps and class-variance-authority to package.json.

2. **HIGH: Missing Toaster render (WR-02):** OrderForm calls `toast()` on success but `<Toaster />` is never mounted. Users get no feedback when placing orders.

3. **HIGH: WebSocket backoff stale closure (WR-01):** `reconnectAttempt` is always 0 in the reconnection delay calculation. The client reconnects every 1 second during outages, never backing off.

4. **HIGH: Currency mismatch (WR-03):** PositionsTable enriches positions with native-currency prices from WebSocket but subtracts INR-based costs. Non-INR asset P&L values are meaningless.

5. **MEDIUM: TypeScript type escape (WR-04):** OrderForm uses `params as any` bypassing PlaceOrderParams validation.

6. **LOW: Settings dialog not closed (IN-05):** Dialog stays open after successful portfolio reset.

None of these issues are deferred to later phases — they must be fixed within Phase 2.5.
