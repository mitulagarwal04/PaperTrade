# Phase 3: Charts & Technical Analysis — UI Design Contract

> Visual and interaction contract for the charts/technical analysis phase.
> Generated by gsd-ui-researcher, verified by gsd-ui-checker.
> Covers requirements TA-01, TA-02, TA-03.

---

## Design System

| Property | Value | Source |
|----------|-------|--------|
| Tool | shadcn/ui | components.json |
| Preset | zinc base, dark mode `"class"`, Inter font, Lucide icons | components.json + tailwind.config.ts |
| Component library | Radix primitives via shadcn/ui | components.json |
| Icon library | Lucide (chart toolbar), TradingView LW built-in (chart UI) | components.json |
| Font | Inter (body), JetBrains Mono or Tabular (numeric/monospace) | tailwind.config.ts |

### Chart Library

| Property | Value |
|----------|-------|
| Library | TradingView Lightweight Charts v4.x |
| Rendering | Canvas/WebGL (not DOM) — chart internals NOT styled via Tailwind |
| Integration | Thin React wrapper component using imperative API + ref pattern |
| Bundle size | ~45KB gzipped |

### Chart Page Architecture

The chart page has two rendering domains:

1. **Shell (DOM)** — Built with React + Tailwind + shadcn/ui. Includes toolbar, control panels, info bar, timeframe selector. This IS covered by this contract.
2. **Chart canvas (Canvas)** — Rendered by TradingView LW internally. Styled via LW API options (color, grid, crosshair). NOT covered by Tailwind. This contract defines the LW API color mapping.

## Spacing Scale

Standard 8-point scale reused from Phase 2.5 (existing Tailwind spacing). Chart page extends with chart-specific zones:

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Toolbar icon gaps, indicator badge padding |
| sm | 8px | Toolbar button padding, timeframe button gap |
| md | 16px | Default spacing between toolbar groups |
| lg | 24px | Section padding, chart container padding, title spacing |
| xl | 32px | Layout gap between toolbar rows and chart |
| 2xl | 48px | Info panel top padding, empty state vertical spacing |
| 3xl | 64px | Page-level top/bottom margins |

### Chart Page Layout Zones (vertical stack, top to bottom)

| Zone | Height/Behavior | Elements |
|------|----------------|----------|
| Title bar | auto (h-10) | Symbol name, current price + change badge |
| Timeframe selector | 36px | 7 preset buttons (1D/5D/1M/3M/6M/1Y/5Y), no wrap |
| Drawing toolbar | 40px (collapsible) | Icons: trendline, horiz, vert, ray, text, undo, redo, delete |
| Indicator toolbar | 36px | "Indicators" dropdown trigger + active indicator chips |
| Chart canvas area | flex-1 (takes remaining space) | TradingView LW canvas container, min-h-[300px] |
| Price info bar | 28px | OHLC values, change %, volume (optional, can be appended to title bar) |

### Exceptions

- **Toolbar buttons**: 32px height (not standard 44px touch targets). These are desktop-priority compact controls. Touch targets on mobile should use 40px minimum (add `min-h-[40px]` breakpoint).
- **Timeframe buttons**: 28px height. Compact enough for 7-across layout.
- **Chart container**: No padding between canvas and container edge. Zero gutter for data.

| Context | Format | Rule |
|---------|--------|------|

| Price change | `+XX.XX / -XX.XX` | Sign prefix always shown, 2 decimals |

| Volume | Compact: `1.2M`, `45.3K` | K/M/B suffixed |

| MACD value | 2 decimals: `0.45` | Float |

## States

### 1. Loading (Pending)

**Visual description:**
- Chart container area shows a skeleton matching the chart aspect ratio (16:9 or 4:3 preferred, responsive)
- Title bar shows symbol text skeleton, price skeleton, and change skeleton
- Timeframe buttons are disabled (opacity-50, pointer-events-none)
- Toolbar is hidden or skeleton (hidden preferred — don't show tools for non-data)

**Skeleton design for chart area:**
```
+------------------------------------------+
|  [========]  [=======]  [====]           |  <- 3 skeleton lines simulating price action
|  /\/\/\/\/\/\/\/\/\/\/\/\/\/\/\/\/\/\    |
| /                                        |
+------------------------------------------+
```
- Implement as a `ChartSkeleton` component using existing `<Skeleton>` from LoadingSkeleton
- Area: `aspect-[16/9]` on desktop, `aspect-[4/3]` or `h-[350px]` on mobile
- Visual: layered horizontal skeleton bars with varying widths to suggest a chart shape
- Use `animate-pulse` (existing) + `motion-reduce:animate-none`

### 2. Error (API Failure)

**Visual description:**
- Chart container renders the existing `ErrorBanner` component (reused exactly from Phase 2.5)
- ErrorBanner positioned at the top of the chart container (inside the border box)
- Below ErrorBanner: empty dark gray area (surface-1 bg)
- No timeframe buttons shown (or disabled — disabled preferred for consistency)
- No toolbar shown

**Component reuse:**
```tsx
<ErrorBanner
  message="Could not load chart data for RELIANCE."
  onRetry={() => refetch()}
/>
```

### 3. Empty (No Data / 0 Candles Returned)

**Visual description:**
- Chart container shows centered content
- Muted icon: `BarChart3` from Lucide (40% opacity, 48x48)
- Heading: "No chart data available for {symbol}" — text-primary, text-base
- Body text: "The symbol may not support charting or no data exists for this timeframe." — text-secondary, text-sm
- Timeframe selector is active (user can try switching timeframes)
- No toolbar drawn (no data to draw on)

```
+------------------------------------------+
|                                          |
|          [BarChart3 icon]                |
|    No chart data available for AAPL      |
|  The symbol may not support charting...  |
|                                          |
+------------------------------------------+
```

**Distinction from Loading:** Empty state has no skeleton animation. It is a static centered layout.

### 4. Real-Time Updating (Live Data)

**Visual description:**
- Chart is fully rendered with OHLC data
- Current (last) candle updates in real-time via WebSocket price ticks
- Candle body/wick extends/retracts as the current period progresses
- No visual "flash" or animation on price update (TradingView LW handles smoothly)
- WsReconnectBanner at the header level handles connection status (existing pattern)
- If WS disconnects: price ticker freezes, but candle chart remains static. WsReconnectBanner shows warning. No chart-specific error state for WS disconnect (TradingView LW can't update but candles stay).

**Price info bar behavior:**
- Current price updates in real-time (from priceStore)
- Price change updates in real-time
- Price color: positive.DEFAULT (#22C55E) when price > previous close, negative.DEFAULT (#EF4444) when price < previous close
- Transition: no animation on price change (instant update preferred for financial data)

### 5. Indicator Toggle (Transition State)

**Visual description:**
- When toggling an indicator on: TradingView LW adds the series immediately (no loading state for client-side indicator calc)
- When toggling an indicator off: TradingView LW removes the series immediately
- No spinner or skeleton needed — indicators are calculated client-side and render in <100ms
- Indicator chip appears/disappears from the active indicators bar immediately

### 6. Drawing Tool (Active State)

**Visual description:**
- When a drawing tool is selected: cursor changes to crosshair (for line tools) or text cursor (for text tool)
- Toolbar button for active tool has `bg-surface-3` background + `ring-1 ring-info/60` ring
- On canvas: click to place line start point, drag to extend, release to finalize
- Drawn elements have visible selection handles (squares on endpoints) when hovered or selected
- Selected drawing: handles visible, line slightly brighter
- Unselected drawing: handles hidden, line at normal opacity

## Interaction Contract

| Interaction | Behavior |
|-------------|----------|
| Click timeframe button | Switch chart timeframe. Button becomes active (bg-surface-3 + font-semibold). Chart reloads data for new interval. Previous indicators re-applied. |
| Click indicator from dropdown | Toggle on/off. Checked = visible, unchecked = removed. Dropdown stays open for multiple selections (use `DropdownMenuCheckboxItem`). |
| Click indicator chip X | Remove indicator from chart immediately. No confirmation. |
| Click drawing tool icon | Activate drawing mode. Tool becomes selected (highlighted). Previous tool deactivates. Click again or press Escape to deactivate. |
| Click on canvas (drawing active) | Place anchor point. Drag to extend. Release to finalize. |
| Click on drawn shape | Select shape. Show handles. Click Delete key or toolbar delete to remove. |
| Drag chart | Pan left/right through time. TradingView LW handles natively. |
| Scroll on chart | Zoom in/out. TradingView LW handles natively. |
| Press Escape | Deactivate current drawing tool. Unselect selected drawing. |
| Resize window | Chart resizes responsively. TradingView LW `chart.resize()` called on container resize (ResizeObserver). |
| Mobile: tap/hold on chart | Crosshair appears with price/time tooltip (TradingView LW touch support). |
